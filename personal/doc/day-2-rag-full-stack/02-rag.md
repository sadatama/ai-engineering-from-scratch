## 学习要点

### 已掌握

1. **RAG vs Fine-tuning 五维对比**：成本/时效性/可审计性/幻觉/数据隐私——FT 唯一胜出场景是"永久改变风格/语气/推理模式"
2. **四步流水线**：Retrieve → Augment → Generate → Answer，每步的数据流清晰
3. **Chunking 甜点区间**：256-512 tokens + 50 token 重叠，太小丢失上下文、太大稀释相关性
4. **离线/在线分离**：索引可离线跑数小时，查询必须亚秒级响应
5. **TF-IDF 原理**：(n+1)/(doc_count+1) 防止除零和平滑稀有词
6. **Memory ↔ RAG 对话历史**：无对话历史的 RAG 对追问"那企业版呢？"无法正确检索

### 需校准

| 概念 | 原回答 | 精确表述 |
|------|------|------|
| `simple_generate` 缺失什么 | "核心的学习与理解的能力" | 精确回答：不能合成新文本，只能从 chunk 中选择已有句子。LLM 生成的三项能力：(1) 跨多 chunk 综合信息 (2) 用自己语言改述 (3) 推断隐含结论 |
| 上下文预算占比 | "1280 tokens 占 50%+" | 1280/2500≈51%（下限），1280/5000≈26%（上限）。剩余被 system prompt + 对话历史 + 用户查询本身占用 |
