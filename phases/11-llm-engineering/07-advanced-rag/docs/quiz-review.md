## 学习要点

### 已掌握

1. **基本 RAG 三种失败场景**：模糊查询 / 多跳推理 / 大规模 ANN 误差——根因是"向量相似度 ≠ 相关性"
2. **混合搜索 + RRF**：语义 + BM25 互补，RRF 通过排名（非原始分数）合并，k=60 防止第一名垄断
3. **Reranking 流水线**：Hybrid top-50 → Cross-Encoder → top-5，延迟约 1000ms
4. **HyDE 价值判断**：弥合"问题空间"和"答案空间"的差距，高精度场景值得 500-2000ms 额外延迟
5. **父子 Chunking ↔ Cascaded Retrieval**：子 chunk 精确匹配 + 父 chunk 提供上下文，与粗筛→精排模式同构
6. **Faithfulness 评估的局限性**：词重叠法在"回答只用 chunk 中少量关键信息"时误判，生产中用 LLM-as-judge
7. **元数据过滤的安全角色**：时间门禁 + 类别门禁，防止过时/无关文档进入检索

### 需巩固（Q8 场景判断）

| 场景 | 错误预测 | 正确方案 | 规律 |
|------|:---:|------|------|
| 精确错误码 "E-4021" | 仅向量 | **BM25 或 Hybrid** | Embedding 可能把错误码当噪声，BM25 对精确字符串匹配完美 |
| 口语化查询 | 仅 BM25 | **向量或 Hybrid** | 口语化表达与文档原文关键词不重叠，需要语义匹配 |
| 200 万文档 | Hybrid+Rerank ✅ | Hybrid+Rerank | 大规模下单一方法有盲区 |

**核心规律**：BM25 擅长精确字符串匹配，向量搜索擅长语义模糊匹配——这正是 Hybrid 存在的理由。

## Advanced RAG 技术栈速查

| 技术 | 作用 | 延迟代价 | 精度增益 |
|------|------|:---:|:---:|
| BM25 | 精确关键词匹配 | 极低 | 补向量搜索的精确匹配盲区 |
| RRF | 合并多路排序列表 | 可忽略 | 自然平衡语义+关键词信号 |
| Cross-Encoder Rerank | 对 top-50 精排 | ~1000ms | 显著提升 top-5 精度 |
| HyDE | 模糊查询改写为假设答案 | 500-2000ms | 模糊查询场景提升大 |
| 父子 Chunking | 小 chunk 检索 + 大 chunk 返回 | 可忽略 | 上下文更完整 |
| 元数据过滤 | 搜索前缩小范围 | 可忽略 | 减少无关结果 + 安全防护 |
