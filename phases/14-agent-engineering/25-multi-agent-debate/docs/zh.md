# 多 Agent 辩论与协作

> Du 等人（ICML 2024，"Society of Minds"）运行 N 个模型实例，独立提出答案，然后在 R 轮中迭代地相互批评，最终收敛。该方案提高了事实准确性、规则遵循能力和推理能力。稀疏拓扑在 token 成本上优于全连接网格。

**类型：** 学习 + 构建
**语言：** Python（标准库）
**前置条件：** 第十四阶段 · 12（工作流模式），第十四阶段 · 05（Self-Refine 与 CRITIC）
**时间：** 约 60 分钟

## 学习目标

- 解释辩论协议：N 个提案者，R 轮辩论，收敛到一个共享答案。
- 描述辩论为何能提高事实准确性、规则遵循能力和推理能力。
- 解释稀疏拓扑：并非每个辩论者都需要看到其他所有人。
- 在脚本化 LLM 上使用标准库实现全连接网格和稀疏变体的辩论；测量 token 成本与准确性。

## 问题

Self-Refine（第 05 课）是一个模型自我批评——存在群体思维的风险。CRITIC（第 05 课）将批评建立在外部工具之上——并非总是可用。辩论引入了第三种模式：多个实例，交叉批评，通过分歧实现收敛。

## 概念

### Society of Minds（Du 等人，ICML 2024）

- N 个模型实例对同一问题独立提出答案。
- 在 R 轮中，每个模型阅读其他模型的提案并进行批评。
- 模型根据批评更新自己的答案。
- R 轮之后，返回收敛的答案。

由于成本原因，原始实验使用 N=3，R=2。在困难问题上（MMLU、GSM8K、国际象棋走法有效性、传记生成），准确率随着 Agent 数量和轮次的增加而提高。

跨模型组合优于单模型辩论：ChatGPT + Bard 一起使用 > 单独使用任何一个。

### 稀疏拓扑

"Improving Multi-Agent Debate with Sparse Communication Topology"（arXiv:2406.11776，2024-2025）表明，全连接网格辩论并非总是最优。稀疏拓扑（星型、环型、Hub-and-Spoke）可以在更低的 token 成本下达到相同的准确率。每个辩论者只看到同伴的一个子集。

影响：

- 全连接网格 N=5，R=3 = 5 × 3 = 15 个提案，每个阅读 4 个同伴 = 60 次批评操作。
- 星型 N=5，R=3（一个中心节点 + 4 个辐条）= 15 个提案，辐条只阅读中心节点 = 12 次批评操作。

### 辩论有帮助的场景

- **事实准确性。** N 个独立提案，交叉检查减少幻觉。
- **规则遵循。** 国际象棋走法有效性——一个模型漏掉某条规则，其他模型捕捉到。
- **开放式推理。** 多种视角框架缩小到正确答案。

### 辩论有损的场景

- **延迟敏感的用户体验。** N × R 轮串行执行的延迟可能无法接受。
- **成本敏感的规模化。** 每个问题 N × R 的 token 消耗。
- **简单事实查询。** 一次查询比五次辩论更便宜。

### 2026 年实际应用实例

- **Anthropic Orchestrator-Workers**（第 12 课）——带综合步骤的辩论变体。
- **LangGraph Supervisor**（第 13 课）——中心路由器 + 专业 Agent 可以将辩论实现为一个节点。
- **OpenAI Agents SDK**（第 16 课）——Agent 之间的来回交接实现迭代批评。
- **多 Agent 评估**——将辩论与 Evaluator-Optimizer 配对以获得评估信号。

### 该模式可能出错的地方

- **收敛崩塌。** 所有 Agent 收敛到第一个错误答案。通过要求分歧轮次来缓解。
- **中心节点故障。** 在星型拓扑中，一个坏的中心节点会污染所有人。轮换中心节点或使用多个中心节点。
- **提示词同质化。** 所有 Agent 使用相同的提示词；它们产生相同的答案。使用多样化的提示词和/或模型。

## 构建

`code/main.py` 使用标准库实现辩论：

- `Debater` 类（脚本化 LLM，具有每个辩论者独立的主张倾向）。
- `FullMeshDebate` 和 `SparseDebate` 运行器。
- 三个问题：一个事实性问题，一个基于规则的问题，一个推理性问题。
- 指标：收敛答案、收敛轮次、总批评操作数。

运行：

```
python3 code/main.py
```

输出：每种协议的准确率和成本；在 2/3 的问题上，稀疏方案以更低的成本达到与全连接网格相同的效果。

## 使用

- **Anthropic Orchestrator-Workers** 用于简单的 2-3 工作者辩论。
- **LangGraph** 用于带检查点的有状态多轮辩论。
- **自定义** 用于研究或专门的正确定性保证。

## 交付

`outputs/skill-debate.md` 搭建一个多 Agent 辩论框架，支持可配置的拓扑、N、R 以及收敛规则。

## 练习

1. 实现"强制分歧"规则：在第 1 轮，每个辩论者必须产生不同的提案。测量对收敛速度的影响。
2. 添加置信度加权聚合：辩论者返回（答案，置信度）；聚合器按置信度加权。这有帮助吗？
3. 将一个"Agent"替换为具有不同意见的另一个脚本化 LLM。异构性是否提高了准确率？
4. 在 3 个问题上测量全连接网格 vs 稀疏方案的 token 成本。绘制成本 vs 准确率的图表。
5. 阅读 Society of Minds 论文。将你的玩具实现移植到 N=5，R=3。什么会出问题？什么会变得更好？

## 关键术语

| 术语 | 人们的说法 | 实际含义 |
|------|----------------|------------------------|
| Debate（辩论） | "多 Agent 批评" | N 个提案者，R 轮交叉批评，收敛 |
| Full mesh（全连接网格） | "每个人阅读每个人" | 每个辩论者每轮阅读所有同伴 |
| Sparse topology（稀疏拓扑） | "有限的同伴视野" | 辩论者只阅读同伴的一个子集 |
| Hub-and-spoke（中心辐射型） | "星型拓扑" | 一个中心辩论者，N-1 个辐条只阅读中心节点 |
| Convergence（收敛） | "达成一致" | 辩论者收敛到一个共享答案 |
| Society of Minds | "Du 等人的辩论论文" | ICML 2024 多 Agent 辩论方法 |
